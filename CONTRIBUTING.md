# Contributing

Thanks for your interest in contributing! Here's how to get started.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/modern-dev-portfolio-template.git`
3. Install dependencies: `npm install`
4. Create a feature branch: `git checkout -b feature/your-feature`
5. Start the dev server: `npm run dev`

## Guidelines

- **Keep it template-friendly.** Changes should benefit all users, not just your specific use case.
- **No hardcoded content.** All personal content belongs in `src/config/portfolio.config.js`.
- **Follow existing patterns.** Match the code style, component structure, and naming conventions already in place.
- **Small, focused PRs.** One feature or fix per pull request.
- **Test on mobile.** Ensure your changes are responsive and mouse effects are disabled on touch devices.

## Pull Request Process

1. Update the README if your change affects setup or configuration.
2. Ensure the app builds without errors: `npm run build`
3. Write a clear PR title and description explaining _what_ and _why_.
4. Reference any related issues.

## Reporting Issues

- Use GitHub Issues for bugs and feature requests.
- Include your browser, OS, and steps to reproduce for bugs.
- Search existing issues before opening a new one.

## Code of Conduct

Be kind, constructive, and respectful. We're all here to build something useful.
